Git and GitHub

What is Git?

Git is an open source version control system. A key word to note is version control system.

What is a version control system.

A version control system tracks and records changes to any file(s) allowing one to recall specific iterations later on as needed.
For example you are an author of a book that you need it published in some months to come. As an author considering the readers you are writing to, you are required to review the writing as much as you can in order to communicate. While doing this you end up to make some changes that alter the original writing. Therefore, you will make another copy of the original book file and make changes. During this process of reviewing, you may need your suggested authors to review and make changes. These changes are recorded based on who made the and you can also have the chance to use the original book after seeing the changes could not fit in. Now, this is made possible through Git.
Git is a version control version for it is responsible to store the project in a repository(file folder),track the file changes and enable code collaboration between teams remotely.

In this example we learn that git:

  1. Tracks file changes.
  2. Tracks who made the changes.
  3. Enables file collaboration with various people (remotely).

Other than git, we have other version control systems which include: Mercurial, AzureDevOps, Bitbucket, just to name a few.
But in this blog, we shall not talk about the examples given above but shall focus on Git and GitHub.

Installing Git

For both Windows and MacOs users Git doesnt come preinstalled in the system but I shall guide you on how to check whether Git is installed and how to install it. For the Windows users you can search for the Command Line and run it. For the Mac Os users, you can use the Terminal. In both Terminal and Command Line, type 

git --version
or you can type 
git -v
